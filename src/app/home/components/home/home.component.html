Tecnologias Usadas

Patrones de Diseño

App Modular

La aplicacion se desarrolo bajo el principio de modularidad. La app esta dividida en tres modulos, el modulo courses que agrupa todas las funcinalidades relacionadas a la consulta de estos. El modulo admin que agrupa las funciones de gestion del contenido. Por ultimo el modulo de home, que es la seccion encargada de comunicar las instrucciones de la prueba.

Lazy Loading

Se aplico el patron de diseño Lazy Load para retrasar la carga o inicialización de un modulo hasta el momento de su utilización. De esta manera dando una respuesta muchas mas rapida en sus cargas iniciales.

Principio de responsabilidad única

Se aplico este patron teniendo en mente futuras actualizaciones. Tener un codigo corto y legible, permite actulizar con alta rapidez. Identificar un error es mucho mas sencillo, ya que el fallo se produce en una unica responsabilidad.

Instrucciones

Gestion del contenido

Ingresa al modulo del administrador, donde pueden crear o eliminar contenidos. Ir como admin

Contenidos vistos

Los contenidos vistos se identifican por el icono de un ojo en la parte superior derecha de cada card. Esto se observa en la seccion de cursos.

Para lograr esta funcion, se guarda el id de cada contenido en un arreglo alojado en el LocalStorage del navegador.

Fecha Limite

Los contenidos tinen una fecha limite para ser consultados. Esta funcion se implento con la libreria moment.js para comparar fechas y actualizar las directivas correspondientes.
